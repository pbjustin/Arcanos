# Documentation Maintenance

> Updated: 2026-03-03

This repo keeps documentation close to the code it describes.

## What to update when refactoring
- **Routes / endpoints** → `docs/API.md`
- **Env vars** → `docs/CONFIGURATION.md` + `.env.example`
- **Deployment behavior** (Railway/start/healthchecks) → `docs/RAILWAY_DEPLOYMENT.md` + `RAILWAY_COMPATIBILITY_GUIDE.md`
- **OpenAI integration** (Requests, Responses, tools, streaming) → `docs/OPENAI_RESPONSES_TOOLS.md`
- **Large refactors** → add a short entry to `CHANGELOG.md` (or a dedicated doc under `docs/` if multi-step)

## Generated docs
Some docs are generated and may be updated by scripts:
- `docs/BACKEND_INDEX.md` (generated by `scripts/reindex-codebase.js`)

If you move or delete files, regenerate the index or apply a small manual patch as a follow-up.

## Deleting docs
Delete documents when:
- the feature/subsystem is removed (e.g. routes, packages)
- the doc duplicates another doc and is no longer maintained

Prefer redirecting (a short note + link to the new doc) for 1–2 releases before deletion when possible.
